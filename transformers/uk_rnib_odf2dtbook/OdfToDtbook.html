Overview

This script will transform an Open Document Text file (*.odt) to DTBook.

Input Requirements

The input Open Document Text file needs to conform to the constraints and authoring style as defined in the Open Document Text to DTBook authoring guide.

Output

A DTBook document.

Configuration

Input ODF file
Required. Select the input *odt file.
Output DTBook file
Required. Select the output DTBook file.
XML Validation Report
Optional. Path to a file to store a validation XML report in.

Open Document Text to DTBook authoring guide

How to write ODF documents that will easily transform into Daisy dtbook documents. A user guide.

The structure of the Daisy dtbook file is directly derived from the styling information, which is derived from the styles you use.

Headings

ODF, like other office tools, has pre-defined levels of headings. In the styles and formatting menu (F11 to access), these are shown as heading 1 through to heading 9. Use these headings. Do not create your own, either explicitly, or derived from the default headings. If that isn't good enough for you, write the Daisy version seperately. Remember that Daisy books aren't as visually pretty as your Office documents, they are meant to pass on information from their content.

Any level beyond heading 6 will be ignored.

The first piece of content in the document must be a heading level 1, as required by dtbook.

Headings must follow the principles set out in dtbook, i.e. after a level 1, the only valid headings are level 2 or another level 1 heading. Don't omit levels. This is required by the transform, and makes sense stylistically.

Block level styling

Block level means the sequence of paragraph level items as the document is scanned top to bottom (in Western usage). If you set to on, the menu item view / non printing characters, control F10, then the presence or absence of the end of paragraph character indicates where the blocks end.

In ODF, it is possible to have content which appears as a new block level item, but in fact is a continuation. For example, a small graphic with text following it visually looks like a new paragraph below the graphic. In fact by default in some office applications, the text is a continuation of the block (paragraph) in which the graphic is located. Place the cursor directly after the graphic and hit return to separate the blocks.
