File signature detector a.k.a Linux Magic.

Detect the type of a resource based on magic numbers (byte signature) or XML tokens (XML signature).

The signature data (signature library) is stored in an XML document (see SignatureLibrary.rng); this package ships with a default exemplar of such a library.

Usage example:


  //create instance, optionally load the default library
  SignatureDetector detector = new SignatureDetector(true);
  //load a user library
  detector.loadLibrary(myLibrary.toURL);
  //detect a resource
  List<Signature> list = detector.detect(myResourceURL);

The return type is a List of Signature objects. This is because many times, a resource will match several signatures. For example (and depending on which libraries are loaded), a Daisy 2.02 NCC file could match an 2.02 NCC signature and a XHTML 1.0 signature. The list is sorted with closest match first.

Documentation on how to compose a signature library (or extending the shipped one) is available in ./SignatureLibrary.rng
