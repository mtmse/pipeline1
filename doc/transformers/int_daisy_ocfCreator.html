Transformer Purpose

Create a Open eBook Publication Structure Container Format (OCF) 1.0 archive from one or several input documents.

Input Requirements

A list (1-n) of documents to add.

If documents to add constitute filesets (such as an OPS publication), only point to the fileset manifest (in the case of OPS, package file).

Output

On success

An OCF 1.0 container

On error

On error, this transformer will send a fatal message, then throw an exception and abort.

Configuration/Customization

Parameters (tdf)

input
File.pathSeparator separated list of path specs
output
Pathspec of output OCF .epub file

Extended configurability

None at time of writing

Further development

Implement IRI validity checking, and further file name constraints as per the OCF spec.
